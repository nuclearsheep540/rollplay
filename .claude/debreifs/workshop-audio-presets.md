## 1. Goals Set

From the plan:

- **Part A — Workshop audio tool**: strip the 6-track DAW back to a per-asset single-track Loop Editor. Delete the custom canvas (`WaveformCanvas.js`), restore WaveSurfer with Regions plugin. Target ~400 lines from 1091.
- **Part B — Preset backend**: `PresetAggregate` in `library/` module. DM-scoped. Slots = `{channel_id, music_asset_id}` only (no overrides — assets own config). JSONB slots column, CRUD commands/queries, REST endpoints.
- **Part C — Workshop preset editor**: new Workshop tile for preset CRUD. Left pane = preset list, right pane = channel-slot grid with `AssetPicker` per slot. Dirty-tracking + save.
- **Part D — In-game dropdown**: wire the placeholder `<select>` at `AudioMixerPanel.js:451-464` to list + load presets via existing `sendRemoteAudioBatch` WebSocket path. Read-only (no authoring in-game).

Core invariant throughout: **single source of truth for audio config stays on `MusicAssetAggregate`**. Presets store only routing, never override.

## 3. Challenges

### 3.1 ConvolutionReverb `destroy()` crashed on teardown
Null reference inside parent `super.destroy()` which called `disconnect()` on nodes already nulled. Fixed by matching the HighPassFilter pattern: disconnect first, then null, then `super.destroy()`. Order of teardown matters in Web Audio — child-level nulling before parent disconnect = null reference blow-up.

### 3.2 Reverb solo/mute silently did nothing
Long-standing latent bug in the game mixer. `engine.updateMuteSoloState()` ran a uniform pass that overwrote the composite-key logic that used to understand `channelId:reverb`. Pointed to commit `673c570` in git history to see what *used* to work. I initially proposed two patches; user pushed back — reverb is not a second-class citizen, it's a channel. Led to the SendChannel refactor.

### 3.3 `duration_seconds` showed "—" in preset editor
I PATCHed duration via `/api/library/{id}/audio-config` as a "quick fix". Harsh user correction: **"why would you change the asset duration in seconds!? the asset hasn't changed, this is all configuration that sits on top of an asset, why would you mutate the truth of the file's metadata?"** Reverted. The real cause was a stale TanStack cache — a hard refresh fixed it. Lesson absorbed (see §L4).

### 3.4 Snap-on-drag fought WaveSurfer's internal drag state
Remove + re-add region each tick caused WaveSurfer's drag controller to lose its handle. Switched to snap-on-release (`region-update-end` event) — simpler, and matches what Reaper/Logic/Ableton do.

### 3.5 Pause + import = old buffer plays through new waveform
Symptom user reported: playhead moves across the new waveform but the audio is the *previous* asset. Root cause: `channel.resume()` replayed `channel._buffer` (stale, never cleared on import). Fixed by `channel.stop()` + `audioBufferRef.current = null` at the top of `importAsset()`.

### 3.6 BPM Detect wiped existing BPM on failure
`setBpm(detected)` was unconditional — if detection returned null, it clobbered the user's manual BPM. Fixed with `if (detected) setBpm(detected)`. Also spotted that `bpm` wasn't in the auto-save effect deps, so manual BPM edits weren't saving.

### 3.7 FastAPI route ordering bug (preset routes shadowed by catch-all)
Starlette routes use first-match-wins. Preset routes were registered after `/{asset_id}` so `/api/library/presets` matched `asset_id="presets"` and returned 404. Moved preset routes *before* `/{asset_id}`.

### 3.8 Grid region-apply effect raced the WaveSurfer build
Region-apply effect fired before the plugin was mounted. Fixed with a `pluginBuildId` state that bumps at the end of the build effect — region-apply, grid-draw, and enable-drag effects all depend on `pluginBuildId`.

### 3.9 Loop drawer "hard toggle" anti-pattern
First cut: the "Loop" drawer opened/closed on a UI-only boolean. User feedback: **"don't hard-toggle a low-level UI component, correctly set the state high-level so everything is enabled correctly."** Refactored so drawer visibility is *derived* from `loopMode` (with a user override flag). State lives at the right level; UI follows.

## 4. Decisions & Diversions

### D1: Preset editor as tab, not Workshop tile (planned tile → shipped tab)

**Plan said:** New top-level Workshop tile "Audio Presets" at `workshop/audio-presets/page.js`.
**Shipped:** Third tab inside the existing Audio Workstation tool (Loop Editor / Mix Editor / Presets), no new route.

**Rationale:** During implementation it was obvious the three concerns are one workflow — edit an asset's loop, assemble assets into a preset, audition the preset. Splitting them into separate Workshop tiles forced DMs to navigate away + back. The Audio Workstation tool becomes a DAW-like multi-view tool (same shape as Photoshop's workspaces).

**Impact:** Fewer files, one `FileMenuBar` reused across tabs, smoother DM UX. No downstream breakage.

### D2: SendChannel as first-class channel (not in plan)

**Plan said:** Nothing about the engine — Part A treated the engine refactor as done.
**Shipped:** Full engine refactor — reverb send registered as a mixer channel with parent cascade, `updateMuteSoloState` unified.

**Rationale:** Fixing the latent reverb solo/mute bug with a patch would have re-entrenched the composite-key design. Making SendChannel a peer of AudioChannel is closer to how real mixing desks model it (a bus is a channel) and eliminates an entire class of bug.

**Impact:** Game mixer's reverb solo/mute now works. Mix Editor gets the same behaviour for free.

### D3: New loop mode `continuous` (not in plan)

**Plan said:** Loop modes: off, full, region. (Plan assumed two looped modes.)
**Shipped:** Four modes — **off, full, continuous, region**. `continuous` = play the intro once, then loop the region (SFZ/SoundFont's `loop_continuous` semantic).

**Rationale:** "Region only" is a harsh cut for tracks with an intro the DM wants played once. Continuous is the musically natural default for most BGM.

**Impact:** Migration `*_rename_loop_mode_region_to_continuous_*.py` renames existing `region` rows to `continuous` where appropriate. `AudioChannel._computeCurrentTimeFromElapsed` re-anchors on region change so the engine handles all four modes from one code path.

### D4: Snap-on-release, not magnetic snap-while-dragging (implementation detail)

**Plan said:** "drag-create and drag-edit are built in" (no snap behaviour specified).
**Shipped:** Snap only fires on the `region-update-end` and `region-created` events. During drag, the region tracks the raw mouse position.

**Rationale:** Matches Reaper/Logic/Ableton. Attempting magnetic snap caused §3.4 (WaveSurfer drag-state fight). Simpler code, expected behaviour.

### D5: WaveSurfer restored after the custom-canvas detour

**Plan said (Context section):** The prior multi-track work removed WaveSurfer (commit `100bb6a`) because it didn't fit the multi-buffer + custom looping design.
**Shipped:** WaveSurfer back. Single-track + Regions plugin eliminates the original incompatibility entirely — there's one buffer, WaveSurfer is pure visual, audio still goes through AudioEngine.

**Rationale:** The custom canvas work (`WaveformCanvas.js`, 260 lines) was load-bearing *only* because the tool was multi-track. Collapsing to single-track made WaveSurfer viable again with ~120 lines of wrapper.

### D6: Preset editor auto-saves instead of Save/Revert

**Plan said:** "Dirty tracking on the edited preset: one `savedSlots` snapshot compared to live state, Save/Revert buttons at the bottom."
**Shipped:** Auto-save on every slot change, no manual Save/Revert.

**Rationale:** The Loop Editor shipped with auto-save and it felt noticeably smoother. Users never *had* to remember to hit Save. The same applies to preset editing — there's no destructive action hidden behind a button, so no reason to gate it.

### D7: Beat/bar grid + time signature (not in plan)

**Plan said:** BPM only.
**Shipped:** BPM + time signature. SVG beat-grid overlay on the waveform (alternating bar shading + beat lines). Bar.beat readouts in the Loop Points drawer. Snap-to-beat toggle.

**Rationale:** Once BPM is set, not drawing a grid against the waveform is a missed opportunity — bar boundaries make loop-point placement obvious. Time signature (stored as `VARCHAR` with a whitelist, not `{numerator, denominator}` columns) keeps the schema simple.

## 5. Current Architecture

### Workshop Audio Workstation (DM-only)
```
/workshop/audio-workstation
  AudioWorkstationTool (shell)
    FileMenuBar — tab-contextual menu
    TabNav: [Loop Editor] [Mix Editor] [Presets]
      Loop Editor (per-asset, single-track)
        WaveformViewer (WaveSurfer + Regions + SVG grid)
        AudioWorkstationControls (loop-points drawer)
        useWorkshopPreview (single channel through AudioEngine)
      Mix Editor (multi-channel preview of a preset)
        VerticalChannelStrip × N (via channelStateAdapter)
        useWorkshopMixEngine (debounced PATCH per asset)
      Presets
        Left pane: preset list, auto-save
        Right pane: slot grid, AssetPicker per channel
        usePresets (TanStack-query CRUD)
```

### Audio Engine (shared by Workshop + Game)
```
AudioEngine
├── AudioContext
├── channels: Map<id, AudioChannel | SendChannel>
│     AudioChannel
│       BufferSource → fader → muteGain → EffectChain → splitter → meters
│                                                                → send taps → SendChannel inputs
│     SendChannel  (first-class, mute/solo/fader/meter)
│       inputBus → fader → muteGain → EffectChain → meter
├── updateMuteSoloState()  — single uniform pass, parent cascade for sends
└── seek/play/stop/pause/loop-mode handling on AudioChannel
```

### Preset data flow
```
Workshop edit ──▶ PATCH /api/library/presets/{id} ──▶ PG (JSONB slots)
                                                        │
In-game load  ──▶ GET /api/library/presets            ◀─┘
              ──▶ GET /api/library/{asset_id}  (per slot)
              ──▶ sendRemoteAudioBatch([load × N])  ──▶ api-game ──▶ MongoDB ──▶ WS broadcast
```

Preset rows never cross the site↔game boundary. The mixer always reads fresh asset config on load, so mid-session edits in the Loop Editor take effect next time the preset is loaded.

## Lessons for Future Claude Instances

These are the concrete course-corrections from this work. They're written for future-me.

### L1: Metadata ≠ configuration. Don't conflate them.
`duration_seconds` describes the audio file itself. `loop_start`, `loop_end`, `bpm`, `time_signature` are configuration *about* how that file plays. Never PATCH metadata through a `/audio-config` endpoint — it's a category error. When a displayed metadata value looks wrong, check the cache before blaming the data.

### L2: Don't rebuild reflexively.
A saved memory now enforces this. Full `npm run build` after a CSS tweak burns 30s and catches nothing the IDE didn't already. Build at end-of-feature or before recommending a commit — not after every edit.

### L3: Understand the symptom before patching.
When the user reports a bug, *talk about it first*. Multiple times this session I jumped to a fix and had to unwind it. Examples: the snap-on-drag fight, the duration PATCH, the mute/solo composite-key patch. The user said it plainly: **"let's chat through the symptom and understand the problem before we start."**

### L4: Check existing patterns before inventing new ones.
The codebase already has:
- A styling system (`colorTheme.js` + Tailwind tokens `surface-*`, `content-*`, `border-*`)
- A set of UI constants (`PANEL_TITLE`, `PANEL_HEADER`, `PANEL_SUB_HEADER`, `PANEL_CHILD`)
- Headless UI wrappers (Modal, Dropdown, TabNav, Combobox)
- Existing nav patterns (e.g. where the Home icon lives in `SiteHeader`)
- Existing components (`AssetPicker`, `VerticalChannelStrip`, `FilterKnob`)

If I find myself picking hex colours, inventing hover states, or wondering where a nav control should go — **stop and grep the codebase**. The pattern already exists.

### L5: Selected = highlighted pill (inverted bg/fg). Disabled = keep colours, change interactivity.
For button groups (loop modes, preset list):
- **Selected:** light background + dark text, stands out on dark surrounding.
- **Default:** dark/muted, hover brightens.
- **Disabled:** keep default colours — cursor-not-allowed + subtext copy signal the disabled state. *Don't* fade the colours.

Got this wrong multiple times. The fix each time was to undo the colour change and just trust the cursor + explanatory subtext.

### L6: "Instead" means replace, not add.
Literal reading of user corrections. If the user says "use the home icon instead" — replace the button with the icon, don't put both.

### L7: Write to state at the right level. Don't hard-toggle low-level UI.
When the Loop drawer's open/closed state was independent of `loopMode`, it was possible to have `loopMode = 'region'` without the drawer showing. Wrong. Derive the drawer visibility from `loopMode` and let a user-override flag handle the edge case. **Set state high, derive UI low** — not the other way around.

### L8: FastAPI route order matters. Specific before catch-all.
`/api/library/{asset_id}` will match `/api/library/presets` if registered first. Always register specific paths before parameterised catch-alls.

### L9: When fixing an old bug, read git history for what used to work.
The user explicitly told me to check commit `673c570` when debugging reverb solo/mute. Git history isn't just audit — it's a record of working designs. The proven code from before a regression is often the best template for the fix.

### L10: "Do X" naming = Linux kernel template-method pattern.
When a base class has `destroy()` that calls `_doDestroy()`, the convention is: framework-level machinery lives in `destroy`, subclass-specific cleanup lives in `_doDestroy`. Same for `connect`/`_doConnect`. Follow it — don't reinvent lifecycle on each subclass.

### L11: Autogenerate migrations, never hand-write.
Enforced by memory. Always `docker exec api-site-dev alembic revision --autogenerate -m "…"`. And if a new model isn't detected, the fix is to add it to `alembic/env.py` imports — not to hand-write the DDL.

### L12: `authFetch`, always. Including inside TanStack `queryFn`/`mutationFn`.
Silent 401s with no refresh path are the failure mode. This session: every new Workshop/Preset hook uses `authFetch`. Only exceptions are the refresh endpoint itself, auth/login pages, public patch notes, and direct S3 PUT.

### L13: `AudioChannel` loop-mode state needs one source of truth for current time.
Splitting "where am I now?" across pause/resume/seek/region-wrap caused the pause + import bug. The fix: `_computeCurrentTimeFromElapsed()` is *the* function, called by every path that needs to know position. Region changes re-anchor, they don't mutate a stored `currentTime` field.

### L14: Pydantic + Python UUIDs: keep UUIDs as UUIDs until serialization boundaries.
From CLAUDE.md but worth re-stating because I nearly forgot in the preset commands. Only stringify inside `EventConfig.data` dicts, HTTP response bodies, or ETL payloads. Method parameters stay as `UUID`.

### L15: The plan is a starting point, not a spec.
The original plan said "new Workshop tile" — shipped as a tab. Said "Save/Revert buttons" — shipped auto-save. Said "BPM only" — shipped BPM + time-sig + grid + snap. Every one of these diversions was right, each was defensible, and each is in Section 4. **Debrief every diversion** — that's what makes the plan + debrief pair useful to future-me.
