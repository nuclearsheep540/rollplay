## Shared Contracts Package

- **NEW:** `rollplay-shared-contracts` pip package — typed Pydantic schemas shared between api-site and api-game
- **NEW:** Contract types for audio (`AudioChannelState`, `AudioTrackConfig`, `AudioEffects`), maps (`MapConfig`, `GridConfig`), images (`ImageConfig`), sessions (`SessionStartPayload`, `SessionEndResponse`), and assets (`AssetRef`)
- **NEW:** 30 contract tests covering round-trip serialization, shape conformance, and constraint validation
- **NEW:** GitHub Actions CI pipeline for contract tests and api-site test suite
- **ENHANCED:** All HTTP session lifecycle payloads (cold→hot, hot→cold) validated by Pydantic at both ends
- **ENHANCED:** WebSocket audio operations construct typed `AudioChannelState` before writing to MongoDB
- **ENHANCED:** SFX channels now included in session audio restoration — previously skipped during ETL

---

## Database Normalization

- **NEW:** `campaign_members` join table replaces `player_ids` and `invited_player_ids` JSONB arrays — supports proper foreign keys, cascade deletes, and role-based queries
- **NEW:** `dnd_classes` lookup table (12 D&D classes) + `character_classes` join table replaces JSONB blob — enables per-class level tracking
- **NEW:** `dnd_abilities` lookup table (6 ability scores) + `character_ability_scores` join table replaces `stats` and `origin_ability_bonuses` JSONB columns — merges origin bonuses into a single row per ability
- **IMPROVED:** Dead columns removed: `campaign.assets`, `campaign.scenes`, `campaign.npc_factory`, `media_asset.session_ids`
- **IMPROVED:** All 5 Alembic migrations include data migration from JSONB → relational with orphaned UUID filtering

---

## Backend Refactoring

- **IMPROVED:** Session endpoints extracted from monolithic endpoint file into dedicated query classes — session history, roster, and dashboard queries now in CQRS pattern
- **IMPROVED:** `StartSession` command decomposed into focused helpers: `_restore_audio_config()`, `_restore_map_config()`, `_restore_image_config()`
- **IMPROVED:** Library aggregates (`MapAsset`, `MusicAsset`, `SfxAsset`) return typed contract objects from `build_*_for_game()` instead of raw dicts
- **IMPROVED:** Hardcoded defaults (volume `0.8`, opacity `0.3`, grid color `#d1d5db`) removed — contract Pydantic defaults are now the single source of truth
- **FIXED:** `GameService._get_active_session()` — return moved inside `try` block to prevent `NameError` on MongoDB connection failure
- **FIXED:** `selectinload` replaces `joinedload` for collection relationships to avoid cartesian product on characters with multiple classes and ability scores

## Test Infrastructure

---

This release is entirely under-the-hood. JSONB columns that stored structured data as opaque blobs have been normalized into proper relational tables with foreign keys, cascade behavior, and lookup tables for D&D classes and abilities. A shared contracts package now enforces typed schemas at the boundary between api-site and api-game, replacing raw dict construction throughout the session lifecycle. A release manifest system provides a committed audit trail of service versions per release.
