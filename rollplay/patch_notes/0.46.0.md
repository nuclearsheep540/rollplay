## ☁️ Fog of War

- **NEW:** Paint and reveal fog directly on top of any map — full DM control over what your players can see
- **NEW:** Brush size slider (2–500 px in mask coords) with paint and reveal modes — soft alpha edges and arbitrary shapes (holes, disconnected blobs, half-transparent regions) all in a single mask
- **NEW:** "Fog of War" section in the in-game DM map panel — paint, reveal, fill, clear, and broadcast updates over WebSocket
- **NEW:** Fog Mask workshop tool — pre-paint a map's fog before a session ever starts, persisted to the map asset
- **NEW:** Atomic mask updates: the entire mask travels in one WebSocket frame as a base64 PNG — no separate fetch, no S3 round-trip, no flicker on slow connections
- **NEW:** Decode-then-swap rendering on the receiving side — the old fog stays on screen until the new mask is fully decoded, so players never see a blank moment

---

## Frontend Architecture

- **NEW:** `app/fog_management/` module — pure-JS `FogEngine`, `useFogEngine` React adapter, WebSocket helpers, and thin UI components, modelled on `audio_management/`. Same module consumed by both the in-game DM panel and the workshop tool with no coupling to either context
- **NEW:** `FogCanvasLayer` slots into `MapDisplay` between the map image and grid overlay — inherits the existing pan/zoom transform automatically
- **NEW:** Engine canvas lives outside React's render cycle — re-renders cannot blank the fog mid-update

---

## Backend & Contracts

- **NEW:** `FogConfig` shared contract (`mask`, `mask_width`, `mask_height`, `version`) — round-trips between api-site, api-game, and the frontend with the same Pydantic validation as `GridConfig`
- **NEW:** `fog_config` JSONB column on `map_assets` (PostgreSQL) — sibling of grid config, carries the persisted mask alongside the map asset
- **NEW:** `PATCH /api/library/{asset_id}/fog` endpoint — atomic full-replace of the fog mask for a map, gated by the same in-active-session 409 guard as the grid endpoint
- **NEW:** `fog_config_update` WebSocket event on api-game — broadcast to all clients in the room after MongoDB persists the new mask
- **NEW:** ETL hot↔cold flow extended for fog — persisted fog hydrates into the live session on start; final fog state writes back to PostgreSQL on session end
- **NEW:** 5 contract tests covering round-trip, defaults, dimension validation, unknown-field rejection, and `MapConfig` composition with fog
